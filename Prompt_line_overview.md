# Prompt Library Overview

## Aim

The aim of this prompt library is to support pharmaceutical manufacturing teams by improving the structure, completeness and review-readiness of GMP-related documentation.

## Flow 1: Quality Event Support

| Prompt | Name | Purpose |
|---|---|---|
| P01 | Quality event initiation and escalation triage | Converts rough event notes into a structured quality event record and flags possible escalation needs |
| P02 | Quality event investigation planner | Creates an investigation plan covering records, people, checks and missing evidence |
| P03 | Root cause and corrective action assistant | Supports structured RCA thinking and corrective/preventive action options |
| P04 | Final quality impact summary drafter | Drafts a quality impact summary for QA review |

## Flow 2: BPR Improvement

| Prompt | Name | Purpose |
|---|---|---|
| P05 | BPR critical-step identifier | Identifies critical process, quality, safety and documentation steps |
| P06 | BPR checklist-format converter | Converts a BPR extract into a simpler checklist-style draft |
| P07 | BPR review-point and change-control checker | Identifies SME, QA, validation, training and document-control review points |

## Flow 3: Product Complaint Support

| Prompt | Name | Purpose |
|---|---|---|
| P08 | Product complaint intake standardiser | Converts complaint details into a structured complaint record |
| P09 | Product complaint processing and investigation planner | Builds a complaint investigation plan |
| P10 | Product complaint recommendation and response drafter | Produces recommended next steps and a safe customer response draft |

## Human oversight

All outputs require human review before use in any formal GMP process.
